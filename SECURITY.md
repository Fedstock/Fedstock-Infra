# Security Policy

This repository is intended to be safe for public portfolio use. It should describe infrastructure structure and deployment practices without exposing deploy-time secrets or account-specific credentials.

## Do Not Commit

The following files or values must not be committed:

- `terraform.tfvars`
- `*.tfstate`, `*.tfstate.*`
- `*.tfplan`
- `.env`, `.env.*`
- AWS access keys or session tokens
- private keys, certificates, or PEM files
- database passwords
- real service credentials
- local-only notes under `notes/`, `codex-notes/`, or `todo.md`

## Secret Handling

- RDS master password is generated by AWS and managed through AWS Secrets Manager.
- ECS task definitions inject the database password as `DB_PASSWORD` from Secrets Manager.
- Application containers should read database connection values from environment variables and secrets, not hard-coded config.
- `terraform.tfvars.example` uses placeholder values only. Real AWS account ids, domain names, ACM ARNs, and bucket names belong in ignored local files.

## Network Boundary

- ALB is the only public ingress point.
- Backend ECS tasks run in private subnets.
- RDS runs in private subnets and is not publicly accessible.
- AI Backend is designed as an internal service called by Backend. It should not be exposed directly to public users in production.

## Pre-Commit Checklist

Before committing:

```bash
git status --short
rg -n "AKIA|SECRET_ACCESS_KEY|BEGIN PRIVATE|password\\s*=|995[0-9]{9}|/Users/" .
terraform -chdir=infra/envs/prod fmt -recursive
terraform -chdir=infra/envs/prod validate
```

If a real account id, private domain, ARN, local path, or password appears in a public-facing document, replace it with a placeholder before committing.
